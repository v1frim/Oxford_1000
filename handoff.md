## Реалізовані фічі

### Режими гри (6 шт)

**Переклад:**
- `🇺🇦 → 🇺🇸` (ua-en) — показ UA → ввод EN
- `🇺🇸 → 🇺🇦` (en-ua) — показ EN → ввод UA

**English only:**
- `📖 Def` (en-def) — показ визначення → ввод слова
- `🔁 Syn` (en-syn) — показ слова → ввод синоніма
- `↔ Ant` (en-ant) — показ слова → ввод антоніма
- `📝 Cloze` (en-cloze) — речення з `____` → заповнити слово

### Гра
- 60 секунд на раунд
- При помилці — correction-панель 3.5 сек з правильною відповіддю, прикладом, TTS
- "Останнє слово" — після таймера ще одна спроба
- TTS через Web Speech API (preferred female US English voice)
- Звукові ефекти: Web Audio API — E5→G5 старт, G5→E5→C5 фініш

### Milestone-вітання (overlay `#milestone-overlay`)
Функція `showMilestone(icon, text, sub, type)` + **черга** `enqueueMilestone({icon,text,sub,type,sfx})`.
Черга показує вітання послідовно (2.7с кожне), щоб вони не перезаписували одне одного.
4 типи (кожен свій колір + звук):
- **milestone** (🎯 жовтий, `sfxMilestone`) — серія ігор за день кратна 4
- **nudge** (💪 синій, `sfxNudge`) — 2-га гра дня (gamesCount % 4 === 2)
- **cleared** (✨ зелений, `sfxCleared`) — усі помилки дня закриті (review)
- **rank** (🏆/🥇 фіолетовий `#c89bff`, `sfxRank` фанфара) — гра потрапила в топ-10 leaderboard.
  Показується ТІЛЬКИ якщо список був уже повний (`scoresWereFull`, ≥10 ігор) — щоб не спамити на старті.
  rank===1 → 🥇 "Новий рекорд!"; rank 2-3 → "топ-3"; 4-10 → "топ-10". Анімація іконки `ms-trophy`.

### Review-режим
- 3 правильні підряд = освоєно
- **TAINTED-логіка:** бодай 1 помилка/пропуск → слово НЕ видаляється з "помилок дня" навіть після 3 правильних
- Таймер рахує скільки пройшло (не зворотний відлік)
- Review direction modal: EN→UA / UA→EN / Випадковий

### Mastery tracking
Категорія визначається **наявністю в `m` (mastery-об'єкті) + значенням streak**, а НЕ лише streak:
- **Знаю** (streak ≥ 3): зелений ✓
- **Вивчаю** — слово вже траплялось у грі хоча б раз (є запис у `m`, streak 0-2): синій ↗
- **Нові** — слово ще НЕ траплялось (немає запису в `m`): оранжевий ●
- ⚠️ Слово зі streak 0 в `m` (відповів НЕправильно/пропустив) = **Вивчаю**, а НЕ Нові.
  Бо "Нові" = `WORDS.length − known − learning` = слова, яких немає в `m` взагалі.
  Код категоризації: `if (s >= 3) known++; else learning++;` — streak 0 свідомо в learning.
- localStorage: `oxford_word_mastery_v1` `{wordIndex: streak}`
- Review-сесії **не** оновлюють mastery (тільки звичайна гра)

### Гра по категоріях (Знаю / Вивчаю / Нові) — додано сесія 10
Три бейджі mastery (`✓ Знаю`, `↗ Вивчаю`, `● Нові`) на стартовому екрані **клікабельні**:
клік → модалка `#cat-modal` зі списком усіх слів цієї категорії (відсортовані за EN,
формат `EN — переклад`, клік по рядку = TTS) + кнопка **«▶ Грати ці слова (N)»**.

- Кнопка запускає **звичайну гру 60 сек, обмежену цими словами** (mastery оновлюється:
  Нові→Вивчаю→Знаю). Це НЕ режим повторень.
- **`startGame(pool, opts)`**: новий другий аргумент. `opts.timed === true` → грати timed-гру
  по кастомному пулу (mastery оновлюється). Без `opts` або з `pool` без `timed` →
  стара поведінка (review-режим: 3× правильно щоб закрити, mastery НЕ оновлюється).
  Розв'язано через дві змінні: `timedPool` vs `activePool`. **Review-флоу не зачеплено.**
- Пул перетинається з валідним пулом активного режиму (`modeValidPool()`): для Def/Syn/Ant/Cloze
  у гру потраплять лише слова, що мають відповідні дані. Якщо перетин порожній → кнопка
  задизейблена «Немає слів для цього режиму».
- Хелпери: `categoryIndices(cat)` ("known"/"learning"/"new"), `modeValidPool()`,
  `openCategoryModal(cat)`, `closeCategoryModal()`, `startCategoryGame()`, конст `CAT_META`.
- Делегований клік на `#mastery-stats` (бейджі мають `data-cat`), переживає перемальовки.
- Клавіші в модалці: **Esc** — закрити, **Enter** — грати; клік по фону / ✕ — закрити.
- ⚠️ Перевірено в сесії 10 через Playwright (headless): обмеження пулу, оновлення mastery
  в обидва боки, перетин з режимом, порожня категорія — усе працює.

**ВАЖЛИВО про "+N ↗" у "Прогресі по днях":** це дельта кількості слів у "Вивчаю"
(скільки нових слів вперше з'явилось за день), а **НЕ** кількість XP-переходів.

**Бонус +3 XP за Нові → Вивчаю (оновлено в сесії 9):** нараховується при **першій в
історії** зустрічі слова в грі — `prev === undefined`, **незалежно** від того, чи
відповідь правильна, неправильна, чи пропуск. Логіка: коли з'являється нове слово,
користувач витрачає час на його вивчення навіть якщо одразу відповів неправильно —
це теж прогрес. `prev === undefined` спрацьовує рівно один раз на слово, тож подвійного
нарахування немає. Код: у `updateMastery` бонус `addXp(3)` стоїть **в обох гілках** —
у `correct` (`prev === undefined && m[wordIndex] === 1`) і в `else` (`prev === undefined`).
Це **узгодило** live-логіку з міграційною формулою `computeDayStatsXp` (`max(0,ΔВивчаю)×3`),
яка й так рахувала будь-який приріст "Вивчаю".

**Історія (сесія 8 — баг повторного +3):** раніше умова була `prev === undefined ||
prev === 0`, через що слово зі streak 0 (скинуте після помилки, але вже «Вивчаю»)
отримувало +3 ЗНОВУ при кожному 0→1. Виправлено на `prev === undefined`. У сесії 9
цей самий захист (`prev === undefined` = рівно раз на слово) збережено — просто бонус
тепер дається й на неправильну/пропущену першу зустріч.

**Про слово "lead":** в словнику є ДВА окремих записи:
- `{en:"to lead", ua:"вести"}` — дієслово
- `{en:"lead", ua:"свинець"}` — іменник (метал, вимова `/ˈlɛd/`)
Це коректно, не об'єднувати.

### XP / Рівні

**Ставки XP:**
| Дія | XP |
|-----|-----|
| Правильна відповідь у звичайній грі | +1 |
| Закрите слово в повтореннях (після 3 правильних) | +1 (одне повторення) |
| Нові → Вивчаю (перше в історії правильне) | +3 бонус |
| Вивчаю → Знаю (streak = 3) | +5 бонус |
| Знаю → Вивчаю (регресія) | −3 |

**⚠️ Важливо:** +3 і +5 — це БОНУСИ на додаток до +1 за правильну відповідь у грі.
Перехід Нові→Вивчаю = +1 + +3 = +4 разом. Перехід Вивчаю→Знаю = +1 + +5 = +6.

**Виправлено в сесії 8 — баг XP за повторення:** раніше `addXp(1)` стояв на КОЖНІЙ
правильній відповіді, у т.ч. у review. Оскільки слово закривається лише після 3
правильних, виходило +3 XP за слово, хоча лічильник 🔁 рахує закриті слова (1/слово).
Тепер у review `addXp(1)` спрацьовує ЛИШЕ коли слово закрите (`reviewCorrectCounts >= 3`)
→ 1 XP = 1 повторення = +1 у 🔁. Звичайна гра без змін (+1 за кожну правильну).

**20 рівнів**, формула порогів: `n*(n+1)/2 * 800` XP (Level 20 = 152,000 XP):
```
1 Новачок 0 | 2 Початківець 800 | 3 Учень 2400 | 4 Практик 4800
5 Знавець 8000 | 6 Фахівець 12000 | 7 Досвідчений 16800 | 8 Майстер 22400
9 Провідний 28800 | 10 Експерт 36000 | 11 Старший Експерт 44000
12 Спеціаліст 52800 | 13 Провідний Спеціаліст 62400 | 14 Лінгвіст 72800
15 Старший Лінгвіст 84000 | 16 Поліглот 96000 | 17 Аналітик 108800
18 Майстер Мов 122400 | 19 Гуру 136800 | 20 Легенда 152000
```

**localStorage ключі XP:**
- `oxford_xp_v1` — загальний XP (число)
- `oxford_xp_day_v1` — `{date: xpGained}` щоденний приріст

**Функції:** `loadXp`, `saveXp`, `addXp(delta)` (також викликає `trackDayXp`), `trackDayXp`, `getTodayXpGain`, `getDayXpGain(date)`, `getLevelInfo(xp)`, `computeRetroXp()`, `renderXpBar()`

**UI:** `#xp-section` під mastery-bar: `⚡ Рівень N · Назва` + зелений піл `+N XP` (якщо > 0 сьогодні) + `N / N XP` дрібно праворуч знизу.

### Duolingo / LingoHut інтеграція

**localStorage ключі:**
- `oxford_duolingo_v1` — `{"YYYY-MM-DD": count}` — кількість сесій Duolingo на день (лічильник скидається щодня)
- `oxford_lingohut_v1` — `{lessonNum: "YYYY-MM-DD"}` — дати проходження уроків LingoHut

**XP за LingoHut:**
```javascript
function getLhXp(n) { return n <= 35 ? 80 : n <= 70 ? 120 : 160; }
```
(Уроки 1-35: 80 XP, 36-70: 120 XP, 71-125: 160 XP)

**Одноразове коригування XP (сесія 5):** Урок 1 LingoHut раніше давав 100 XP → тепер 80.
Міграційний прапор `oxford_lh_adj_v1` вже застосований, -20 XP відкориговано. Не чіпати.

**Перерахунок XP з формули (сесія 8):** прапор `oxford_xp_formula_v2`. Одноразово
перебудовує `oxford_xp_v1` (total) і `oxford_xp_day_v1` (per-day) детерміновано зі
статистики через `computeDayStatsXp()`: за день = `correct×1 + reviewCorrect×1 +
max(0,ΔВивчаю)×3 + (ΔЗнаю≥0 ? ×5 : ×3) + Duolingo×30 + LingoHut`. Це прибрало історичне
перенарахування (баги вище) і ДОБІЛО XP за ранні дні, коли трекінгу XP ще не було
(back-fill). Замінив старий ретро-блок 27–30 травня (`oxford_xp_day_retro_v1`). Міграція
**ідемпотентна** (читає лише статистику), прапор ставиться ЛИШЕ при успіху (throw →
повтор на наступному завантаженні). Далі total і per-day тримаються в синхроні через
`addXp` (saveXp + trackDayXp).
- ⚠️ `v1` → `v2`: справжня причина, чому v1 встановився, а XP лишився старий —
  у коміті `eec07b4` випадково **видалено `const XP_DAY_KEY = "oxford_xp_day_v1"`**.
  Через це `XP_DAY_KEY` стала неоголошеною, і `localStorage.setItem(XP_DAY_KEY,...)`
  у міграції кидало `ReferenceError` ДО `saveXp(_total)` → total не оновлювався, прапор
  (у v1 — поза try) встановлювався. Те саме ламало `trackDayXp`/`getDayXpGain` →
  per-day `⚡` зникло. Константу повернуто (рядок ~9085, з коментарем «do NOT remove»).
  ⚠️ **НІКОЛИ не видаляй `const XP_DAY_KEY`** — від неї залежить увесь per-day XP.

**Функції:** `loadDuoSessions()`, `saveDuoSessions()`, `loadLhDone()`, `saveLhDone()`, `getLhXp(n)`

## Picture Mode (пілот ВІДКОЧЕНО — сесія 4)

⚠️ **Пілот був реалізований і відкочений** (revert коммітів 4288928 + 369de89).
Причина: end screen вперто показувався порожнім (тільки кнопки "Ще раз"/"Меню",
без заголовка і списків слів). Дебаг наосліп без консолі браузера не дав результату,
синтаксис JS був чистий → проблема в runtime/CSS, яку не вдалось відтворити в cloud.

**Якщо повертатись — спершу з'ясувати причину через консоль браузера (F12).**
Можливі підозри: зображення Wikimedia не вантажились (403/CORS) → JS-виключення;
або overlay-CSS ховав контент. Підхід нижче зберігаю як референс.

### Ідея
Новий режим гри: показується фото → гравець вводить слова що бачить → кожне правильне слово зникає → в кінці показуються пропущені з перекладом.

### Технічна реалізація (узгоджена)
- **Зображення:** URL з Wikimedia Commons (стабільні, без hotlink-блокування)
- **Вплив на файл:** мінімальний (~2 КБ тільки URL-рядки)
- **Слова:** pre-defined список Oxford 1000 слів для кожного фото (15-30 слів)
- **Таймер:** 90 секунд
- **Пілот:** 5-10 картинок

### Структура даних (планована)
```javascript
const PICTURE_SCENES = [

    url: "https://upload.wikimedia.org/...",
    title: "На пляжі",

  },
  // ...
];
```

- Новий режим кнопка на стартовому екрані: `🖼 Фото`

## Правило додавання нових слів у словник

**Обов'язковий чеклист при додаванні нового слова:**
1. `WORDS` array — `{en:"...", ua:"..."}` (ua може бути масивом для альтернатив)
2. `TRANS` — IPA транскрипція
3. `EXAMPLES` — англійське речення
4. `EXAMPLES_UA` — переклад речення
5. `EN_TO_UA` — hover-переклад (якщо не додано раніше)
6. Якщо в прикладному реченні є слово якого **немає в WORDS** → додати його теж повністю (пп. 1-5)
7. Якщо слово є в WORDS але немає в `EN_TO_UA` hover → додати тільки EN_TO_UA запис

**⚠️ ПРАВИЛО (сесія 11, за запитом користувача):** будь-яке нове слово, до якого
**ще не було перекладу** (тобто його немає ні в `WORDS`, ні в `EN_TO_UA`), додаємо
**одразу повноцінним ігровим словом** за повним чеклистом (пп. 1-5) — з прикладним
реченням. НЕ обмежуватись лише hover-перекладом (`EN_TO_UA`). Hover-only залишаємо
тільки для слів, які фізично не мають сенсу як окрема ігрова одиниця (рідкісні власні
назви тощо) — але за замовчуванням робимо повноцінне слово.

**Санітарна перевірка після правок:** `node -e "new Function(...)"` на JS-блоці
