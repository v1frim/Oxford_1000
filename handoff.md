## Поточний стан (зроблено)

### Словник
- **1500+ слів** з UA-перекладами
- **Полісемічні слова** мають окремі записи: `{en:"light", ua:"легкий"}` + `{en:"light", ua:"світлий"}`
- Кожне слово має **речення-приклад EN + переклад UA**
- IPA-транскрипція (зберігається, але не відображається)

### Ігровий процес
- Два режими: **UA→US** і **US→UA**
- **60 секунд** на раунд (звичайна гра)
- При помилці — панель з правильною відповіддю на 3.5 сек (або Enter)
- "Останнє слово" — після завершення таймера можна відповісти на поточне слово

### Режим повторення (review)
- Правило **3х правильних відповідей** для освоєння слова
- 60-секундний таймер триває як страховка; після 0 показує `+N` червоним і не зупиняється
- Закінчується коли всі слова освоєні або користувач тисне "Закінчити"

### TTS (озвучка)
- Web Speech API (`window.speechSynthesis`)
- Обирається кращий жіночий голос (Google US English / Microsoft Neural / Samantha)
- 🔊 кнопка для слова і для речення окремо
- Клік на слові в реченні (через hover-translations) також озвучує

### Hover translations у реченнях
- Кожне слово в EN-реченні обгорнуте в `<span class="hw">`
- При наведенні — переклад (з нашого словника + неправильні форми)
- При кліку — TTS того слова
- Стоп-слова (the, a, is, can, ...) пропускаються

### Рейтинги (Leaderboard)
Три вкладки в боковій панелі:
1. **🏆 Найкращі ігри** — топ-10 окремих ігор
2. **📅 Найкращі дні** — топ-10 днів (сума за день)
3. **🔥 Серії днів підряд** — топ-10 streaks

- Топ-3 у кожній вкладці виділені золото/срібло/бронза з 🥇🥈🥉
- Сьогоднішня дата виділена жовтуватим
- Окремий рядок під лінією для "найкращої сьогодні" (★) і "останньої" (ост)
- Постійна синя смужка зліва для останньої гри в топ-10

### Сьогоднішні помилки
- Кнопка `📒 Помилки за сьогодні (N)` на стартовому екрані
- Якщо сьогодні немає — показує `за вчора` або `за DD.MM`
- Слово видаляється з списку лише після **3 правильних відповідей у review** (повне освоєння)

### Гарячі клавіші
- **Enter** — submit / advance correction / start game / restart
- **Пробіл** — speak word (correction panel), start review (start/end screen)
- **Esc** — end game (Закінчити)
- **Tab / Shift+Tab** — cycle leaderboard tabs
- **Left/Right Shift** — cycle modes on start screen

## Pending Work: EN→EN режими

> Узгоджено з користувачем. Перейти на Sonnet 4.6 Medium для імплементації.

### Режими (5 шт)

1. **Definition → Word**
   Show: definition. Expect: word.
   *"a person who teaches at a school"* → **teacher**

2. **Word → Synonym**
   Show: word. Expect: synonym(s).
   *"huge"* → **enormous / massive / giant**

3. **Word → Antonym**
   Show: word. Expect: antonym.
   *"happy"* → **sad**

4. **Cloze (fill in blank)**
   Show: sentence with ____. Expect: missing word.
   *"The cat is sleeping on the ____"* → **sofa**
   **Дані вже є — використати EXAMPLES.**

5. **Contextual definition** (об'єднати з #1)
   Розширена форма definition з контекстом.

### Джерело даних: Free Dictionary API

- URL: `https://api.dictionaryapi.dev/api/v2/entries/en/{word}`
- Безкоштовний, без ключа, без rate-limit (але треба бути ввічливим: ~5 req/sec)
- Покриття: definitions ~95%, synonyms ~70%, antonyms ~50%
- Fallback для відсутніх синонімів/антонімів: пропускати слово для цього режиму

### План імплементації

**Етап 1: Збір даних** (~20 хв виконання скрипта)
1. Створити `fetch-definitions.js` (Node.js скрипт)
2. Прочитати WORDS з index.html, витягти список англійських слів
3. Для кожного: GET до API, парсити definitions/synonyms/antonyms
4. Зберегти в `vocabulary-data.json` (тимчасовий файл)

**Етап 2: Вшити дані в index.html**
- Додати три нові константи: `DEFINITIONS`, `SYNONYMS`, `ANTONYMS`
- Розмір файлу зросте до ~400-500 КБ — норм для GitHub Pages

**Етап 3: Розширити вибір режиму**
Поточний UI: дві кнопки `UA→US` / `US→UA`.
Потрібно: 6 режимів (2 існуючих + 4 нових).
Варіанти:
- Pill row (всі 6 в один ряд)
- Dropdown
- Tabs (як leaderboard)
**Узгодити з користувачем.**

**Етап 4: Ігрова логіка для кожного режиму**
- Адаптувати `nextWord()` для роботи з відповідним джерелом даних
- Адаптувати `isCorrect()` для перевірки синонімів/антонімів/definitions
- Cloze: окремий шаблон рендерингу слова (показ речення з ____)

**Етап 5: UI деталі**
- Підказка на стартовому екрані: "Перекладіть" → залежно від режиму
- Іконка/мітка режиму в рейтингу (зараз "UA→US"/"US→UA"; додати "EN-def", "EN-syn" тощо)
- Чи об'єднувати рейтинги між режимами? — узгодити з користувачем

**Етап 6: Інтеграція з рейтингом**
Питання: чи зберігати окремий топ-10 для кожного режиму, чи спільний з міткою?
**Узгодити з користувачем перед імплементацією.**
