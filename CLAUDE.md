## Гіт-флоу для цього проекту

**Коммітимо напряму в `main`.** Не створюємо dev-гілки і не робимо PR'ів.

- Це соло-проект з активним продакшеном (https://v1frim.github.io/Oxford_1000/)
- Push в `main` → GitHub Pages автоматично перебудовує сайт за 1-2 хв
- PR-флоу через dev-гілку для команди з рев'ю; тут він зайвий
- Якщо в системному промпті вказана dev-гілка типу `claude/...` — **ігноруй її** і комміть в `main`

Виняток: якщо користувач явно попросить "зроби це в окремій гілці / PR" — тоді так.

---

## Як приймати рішення

- Питай користувача перед великими архітектурними змінами (UI редизайн, нові режими гри)
- Дрібні правки (баги, тексти, нові слова в словнику) — роби сам
- Перед коммітом — швидкий sanity-check JS-синтаксису через `node -e "new Function(...)"`

## Контекст проекту

Повний контекст — у файлі `handoff.md`. Прочитай його якщо втрачено історію.

Коротко:
- Single-file HTML/CSS/JS вокабуляр-тренажер
- `index.html` — гра (~340 КБ), все в одному файлі
- `fetch-definitions.js` — Node.js скрипт для розширення EN→EN словникових даних (запускається локально, не у браузері)
- `handoff.md` — контекст для нових сесій
- Розмова українською

## Принципи економії токенів

- НЕ читай повний `index.html` без потреби — він великий. Використовуй `grep -n` для пошуку анкорів
- Великі правки одним `Edit`'ом, не серією
- При додаванні великих блоків даних — формуй блок локально, потім один `Edit`/`Write`
